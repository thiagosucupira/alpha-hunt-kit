# Research protocol

Alpha Hunt is not a grid-search machine. It is a discipline loop.

## Daily loop

1. Source: read papers, notes, prior runs, or new market observations.
2. Score: rank seeds against current board and graveyard.
3. Falsify: write the kill criteria before touching code.
4. Cheap test: run one bounded experiment with fixed config.
5. Distill: record promote, block, or kill.
6. Compound: update state so the next run starts smarter.

## Valid hypothesis shape

A hypothesis must include:

- thesis: what should happen,
- why now: why this seed deserves oxygen,
- observable: which data field proves or falsifies it,
- leakage block: why future data is not leaking into the decision,
- cost model: what friction is included,
- kill criteria: what result ends the idea,
- cheapest next test: the smallest artifact-producing experiment.

## Default verdicts

- PROMOTE: beats current champion and passes stated constraints.
- BLOCK: cannot be tested honestly with available data.
- KILL: fails gate, repeats graveyard entry, or only works after retuning.

The default outcome should be KILL. Survival is earned.
